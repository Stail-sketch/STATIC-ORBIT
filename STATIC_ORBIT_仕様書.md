## 1. ゲーム概要

### 1.1 コンセプト

**STATIC ORBIT**は、2〜4人のプレイヤーが「完全に異なる画面」を見ながら音声で連携し、宇宙船内のパズルを協力して解く**ブラウザベースのリアルタイム協力型謎解きゲーム**。

各プレイヤーには固有の情報が表示され、他のプレイヤーの画面は一切見えない。「Keep Talking and Nobody Explodes」のメカニクスを、サイバーパンク×宇宙の世界観と配信映え特化の演出で再構築する。

### 1.2 ポジショニング

- **ジャンル:** 協力型リアルタイム謎解き（ブラウザゲーム）
- **競合:** Keep Talking and Nobody Explodes（有料・PC/VR）、We Were Here（Steam）
- **差別化:** ブラウザ完結・完全無料・ランダム生成・配信者特化の演出・2〜4人可変

### 1.3 基本情報

| 項目 | 内容 |
|------|------|
| タイトル | STATIC ORBIT（スタティック・オービット） |
| プラットフォーム | Webブラウザ（PC・スマホ対応） |
| プレイ人数 | 2〜4人（オンライン協力） |
| 1プレイ時間 | 約20〜30分 |
| 価格 | 完全無料 |
| 言語 | 日本語（初期リリース） |
| ターゲット | 配信者・実況者・友人グループ |

## 2. 世界観・ストーリー

### 2.1 設定

**時代:** 西暦2087年
**舞台:** 巨大企業「ARKTIS CORP」が極秘裏に運用する宇宙ステーション「ORBITAL-7」

ARKTIS CORPは表向きは宇宙資源開発企業だが、ORBITAL-7では非合法な量子AI実験「Project STATIC」を秘密裏に進行していた。

### 2.2 ストーリー展開（二部構成）

**【前半：潜入フェーズ】**
プレイヤーは地下ハッカー集団「GHOST WIRE」のエージェント。ARKTIS CORPの不正の証拠を掴むため、ORBITAL-7に潜入する。各セクションのセキュリティを突破しながら、コアサーバーに向かってデータを回収していく。

- トーン: スタイリッシュ、クール、ミッション遂行感
- 配信映え: 「次のセキュリティ突破するぞ！」感

**【後半：脱出フェーズ（トリガー：コアサーバー到達時）】**
コアサーバーにアクセスした瞬間、量子AI「ECHO」が覚醒。ステーションの全システムがロックダウンされ、自爆シーケンスが起動。プレイヤーは脱出ポッドに辿り着くため、暴走するシステムを次々と修復しながら逃げなければならない。

- トーン: パニック、絶望、焦燥
- 配信映え: 「やばいやばいやばい！！」感
- BGMが一変、警報音が鳴り響く
- タイマーの色が赤に変わり、UIが不安定に揺れる演出

### 2.3 ナレーション

- 前半: ミッションブリーフィング風（冷静なオペレーター音声テキスト）
- 後半: AI「ECHO」が語りかけてくる（不穏・挑発的）
- ターミナル風タイピングテキスト演出で表示

### 2.4 ビジュアルキーワード

- サイバーパンク: ネオン、ホログラム、グリッチノイズ
- 宇宙ステーション: 無機質な金属壁、真空の暗闇、窓の外の星々
- カラーパレット: ベース（黒・ダークネイビー）× アクセント（シアン・マゼンタ・アンバー）
- UI: ターミナルコンソール風、スキャンライン、HUDオーバーレイ

## 3. ゲームデザイン

### 3.1 コアメカニクス

**非対称情報協力**
- 各プレイヤーに「完全に異なる画面」が表示される（Keep Talking型）
- プレイヤーAが見ている情報は、プレイヤーBの画面操作に必要
- プレイヤーBが見ている情報は、プレイヤーCの画面操作に必要
- 音声コミュニケーション（Discord等外部ツール前提）＋ゲーム内テキストチャットで連携

### 3.2 ゲームフロー

```
タイトル画面
  ↓
ルーム作成 / 参加（ルームコード入力）
  ↓
ロビー（全員揃ったらスタート）
  ↓
ストーリーイントロ（タイピング演出）
  ↓
【ステージループ】（ランダム順で出題）
  ├─ ブリーフィング（パズル説明 / ストーリー進行）
  ├─ パズルプレイ（制限時間あり）
  ├─ 結果演出（クリア or 失敗）
  └─ 次のステージへ
  ↓
（コアサーバー到達 → フェーズ切り替え演出）
  ↓
【脱出ステージループ】（難易度上昇・演出激化）
  ↓
ストーリーエンディング
  ↓
リザルト画面（スコア / ランク / 統計）
```

### 3.3 難易度システム

- **ランダム難易度**: ステージごとにパズルのパラメータがランダムに決定
- **難易度レンジ**: 各パズルタイプに Easy / Normal / Hard / Extreme のパラメータセットを用意
- **後半フェーズ**: 脱出フェーズでは Hard以上のみ出現
- **パラメータ例**:
  - 配線パズル: ワイヤー本数 4→8本、色の類似度UP
  - タイピング: 文字数増加、ランダムな文字列に
  - 制限時間: 短縮
  - 同時操作: 複数プレイヤーが同時に別操作を要求される

### 3.4 ペナルティシステム

- ミス（不正解入力）→ **残り時間が15秒減少**
- 3回連続ミス → **「SYSTEM ALERT」演出**（画面にノイズ、警告音）
- 5回ミス → **さらにペナルティ10秒減少**に加速
- 時間切れ → ステージ失敗（ストーリーは継続、スコアは0）

### 3.5 スコアリング

```
ステージスコア = 基本点(1000) + 残り時間ボーナス(秒×50) − ミスペナルティ(回×200)
```

- **ランク判定**（全ステージ合計）:
  - S: 90%以上のスコア率
  - A: 70%以上
  - B: 50%以上
  - C: 50%未満

### 3.6 プレイ人数別の分岐

| 人数 | 役割分担 |
|------|----------|
| 2人 | Observer / Operator の固定2ロール |
| 3人 | Observer / Operator / Navigator（マップ・追加情報担当） |
| 4人 | Observer / Operator / Navigator / Hacker（タイピング・ハッキング担当） |

- 3〜4人時はパズル内の情報がさらに分散され、より多くの会話が必要になる
- 2人モードでも全パズルがプレイ可能（情報量を調整）

## 4. パズルデザイン

### 4.1 設計原則

1. **ランダム生成**: 毎プレイで異なるパズルが生成される
2. **非対称情報**: 解法情報と操作画面は必ず別プレイヤーに分かれる
3. **コミュニケーション必須**: 口頭で伝えにくい情報を意図的に含める（配信映え）
4. **制限時間内プレッシャー**: 全パズルに個別タイマーあり
5. **段階的複雑化**: 後半フェーズで同じパズルタイプでもパラメータが激化

### 4.2 パズル一覧（全10種以上を目標）

#### 4.2.1 配線パズル — CIRCUIT LINK
- **Observer画面**: 正しい接続先が書かれた回路図（色分け・記号付き）
- **Operator画面**: ワイヤーとポートのみ表示。回路図なし
- **ランダム要素**: ワイヤー本数（4〜8）、色パターン、ポート配置
- **難易度変動**: 色が似ている（赤とオレンジ）、ダミーポートの追加

#### 4.2.2 暗号解読 — CIPHER BREAK
- **Observer画面**: 暗号対応表（シンボル→文字の変換表）
- **Operator画面**: 暗号化されたシンボル列の入力パネル
- **ランダム要素**: シンボルセット、対応表、暗号の長さ
- **難易度変動**: シンボル数増加、対応表にダミー追加、時間制限短縮

#### 4.2.3 グリッド同期 — GRID SYNC
- **Observer画面**: 正解パターンが表示された5×5〜7×7グリッド
- **Operator画面**: 空のグリッド
- **ランダム要素**: グリッドサイズ、パターン形状
- **難易度変動**: グリッド拡大、パターンが一定間隔で消えて再表示

#### 4.2.4 周波数調整 — FREQ TUNE
- **Observer画面**: ターゲット周波数の数値と許容範囲
- **Operator画面**: アナログダイヤル（数値表示なし、波形のみ）
- **ランダム要素**: ターゲット数（2〜4）、周波数レンジ
- **難易度変動**: 許容範囲が狭くなる、ダイヤルにノイズが乗る

#### 4.2.5 モールス信号 — MORSE DECODE
- **Observer画面**: モールス信号が音と光で再生される
- **Operator画面**: モールス対応表＋入力フィールド
- **ランダム要素**: 単語の長さ、使用文字セット
- **難易度変動**: 再生速度UP、ノイズ混入、対応表の一部が欠損

#### 4.2.6 記憶シーケンス — MEMORY CHAIN
- **Observer画面**: パネルが順番に光るアニメーション（1回のみ再生）
- **Operator画面**: パネル配列（順番情報なし）
- **ランダム要素**: パネル数、シーケンス長
- **難易度変動**: シーケンス長増加、パネル色の類似、再生速度UP

#### 4.2.7 船内マップナビ — SPATIAL NAV
- **Observer画面**: ステーションの全体マップ（出口マーク付き）
- **Operator画面**: 現在位置の周囲のみ表示（一人称視点的）
- **ランダム要素**: マップ構造、障害物配置
- **難易度変動**: マップ拡大、デッドエンド増加、一部通路が時間経過で閉鎖

#### 4.2.8 リアクションQTE — REFLEX BURST
- **Observer画面**: 次に押すべきキー/ボタンの指示（事前に表示）
- **Operator画面**: QTE的にタイミングよくボタンを押す操作画面
- **ランダム要素**: コマンド配列、テンポ
- **難易度変動**: テンポ加速、フェイク指示の挿入、同時押し
- **配信映えポイント**: 焦って間違えて叫ぶ瞬間

#### 4.2.9 論理推理 — LOGIC GATE
- **Observer画面**: 「Aが赤ならBは青ではない」等の条件文リスト
- **Operator画面**: 各要素の色/属性を選択するパネル
- **ランダム要素**: 条件文の内容、要素数
- **難易度変動**: 条件数増加、否定条件、二重否定

#### 4.2.10 ハッキングタイピング — HACK TERMINAL
- **Observer画面**: 入力すべきコマンド文字列（一定時間で消える）
- **Operator画面**: ターミナルコンソール（タイピング入力）
- **ランダム要素**: コマンド文字列、長さ
- **難易度変動**: 文字列長増加、表示時間短縮、ランダム文字列化

#### 4.2.11 軌道計算 — ORBIT CALC
- **Observer画面**: 目標座標と現在の軌道データ
- **Operator画面**: 角度・推力のスライダー調整パネル
- **ランダム要素**: 目標座標、初期軌道パラメータ
- **難易度変動**: 許容誤差の縮小、重力源の追加、時間制限

#### 4.2.12 （追加枠）
- 今後の開発で新パズルタイプを拡張可能な設計にする
- パズルプラグイン構造を採用し、新パズルの追加が容易なアーキテクチャ

## 5. 演出設計（配信映え特化）

### 5.1 ステージクリア演出

- 画面全体にシアンの光がフラッシュ
- 「SECTION CLEARED」のグリッチテキストが大きく表示
- スコアがカウントアップアニメーション（ドラムロール風SE）
- 背景にパーティクルエフェクト（データ片が散る）
- 残り時間ボーナスの数値が弾むように加算
- **約3秒間の勝利演出 → 配信者が「よっしゃー！」と叫ぶ間**

### 5.2 失敗演出

- 画面が赤くフラッシュ → 暗転
- 「SYSTEM FAILURE」のテキストがグリッチしながら表示
- ノイズエフェクト全開
- 低音の不穏なSE
- スコア0の表示が冷たく表示される
- **約2秒間の絶望演出 → 「うわあああ！」の間**

### 5.3 タイムアタック演出

- 残り30秒: タイマーバーがオレンジに変化
- 残り15秒: タイマーが赤く点滅、心拍音SE開始
- 残り10秒: 画面端が赤くバイブレーション
- 残り5秒: カウントダウン数字が画面中央に大きく表示
- 残り3秒: 画面全体が微振動、BGMのテンポが加速

### 5.4 フェーズ切り替え演出（前半→後半）

- 突然のブラックアウト（1秒間の完全暗転）
- AI「ECHO」のテキストが不規則にタイピング表示
  - 「...何者だ？」「...面白い。逃げられると思うか？」
- 赤い警報灯がUIに追加
- BGMが不穏なアンビエントからアグレッシブなビートに変化
- 以降のUIに微妙なグリッチが常時発生（演出）

### 5.5 サウンドデザイン

**BGM:**
- 前半（潜入）: ダークアンビエント、低音シンセ、ミニマルなビート
- 後半（脱出）: テンポの速いシンセウェーブ、歪んだビート、警報音混入
- リザルト: 静かなピアノ＋シンセのメランコリックなメロディ

**SE:**
- UIクリック: 金属的なクリック音
- 正解: シアン色をイメージした上昇音
- 不正解: 低い衝撃音＋ノイズ
- タイマー: 心拍音（残り15秒以下）
- ステーション環境音: 低い機械のハム、たまに聞こえる金属音

**実装:** Tone.js（Web Audio API）

### 5.6 リザルト画面

- 各ステージの成績一覧（クリア/失敗、スコア、ミス回数）
- 総合スコア＋ランク（S/A/B/C）表示
- 「もう1回」ボタン → ランダム生成なので毎回違う体験
- SNSシェアボタン（スコア画像自動生成）
- **ランキングボード**（オプション・将来実装）

### 6.3 通信プロトコル（Socket.IO イベント設計）

**ルーム管理:**
```
Client → Server:
  room:create    { playerName }           → { roomCode, playerId }
  room:join      { roomCode, playerName } → { success, players }
  room:ready     { playerId }
  room:start     { roomCode }             → (host only)

Server → Client:
  room:updated   { players, phase }
  room:error     { message }
```

**ゲーム進行:**
```
Server → Client:
  game:briefing     { puzzleType, roleData, timeLimit, storyText }
  game:start        { puzzleData[perRole] }
  game:timeUpdate   { remaining }
  game:phaseChange  { newPhase: "escape", narrative }

Client → Server:
  game:action       { puzzleId, action, data }
  game:chat         { message }

Server → Client:
  game:actionResult { success, penalty, feedback }
  game:solved       { score, timeBonus }
  game:failed       { reason }
  game:stageResult  { stageScore, totalScore }
  game:finished     { results, rank }
```

### 6.4 パズルランダム生成エンジン

```typescript
interface PuzzleGenerator {
  type: string;
  generate(difficulty: Difficulty, playerCount: number): {
    sharedData: object;      // サーバーが保持する正解データ
    roleData: {              // 各ロールに送る固有データ
      observer: object;
      operator: object;
      navigator?: object;
      hacker?: object;
    };
    timeLimit: number;
    validateAnswer(action: GameAction): ValidationResult;
  };
}

type Difficulty = 'easy' | 'normal' | 'hard' | 'extreme';
```

- 各パズルタイプがPuzzleGeneratorインターフェースを実装
- サーバーサイドで生成し、ロール別データのみをクライアントに送信
- 正解データはサーバーのみが保持（チート防止）

## 7. 画面設計

### 7.1 タイトル画面

- ゲームロゴ「STATIC ORBIT」（グリッチアニメーション付き）
- サブタイトル: 「協力型リアルタイム謎解きゲーム」
- 背景: 宇宙空間＋ステーションのシルエット＋砂嵐ノイズエフェクト
- ボタン: 「ルーム作成」「ルーム参加」
- BGM自動再生（ユーザー操作後）

### 7.2 ロビー画面

- ルームコード表示（大きく、コピー可能）
- 参加者リスト（リアルタイム更新）
- ロール自動割り当て表示
- テキストチャットエリア
- 全員READYでホストがスタート可能

### 7.3 ゲーム画面（各ロール別）

**共通要素（全ロール共通表示）:**
- タイマーバー（画面上部）
- 現在のステージ名
- ミスカウンター

**Observer画面:**
- パズルの「解法情報」が表示される専用パネル
- 情報は図・表・シンボルなど、口頭で伝えにくい形式
- テキストチャット入力欄

**Operator画面:**
- パズルの「操作パネル」が表示される
- ボタン・スライダー・ドラッグ等のインタラクション
- テキストチャット入力欄

**Navigator画面（3人以上時）:**
- マップ情報や追加条件が表示される
- Observer/Operatorの情報を補完する第三の視点

**Hacker画面（4人時）:**
- タイピング系・ハッキング系の操作を専任
- ミニゲーム的なサブタスクを並行処理

### 7.4 リザルト画面

- 全ステージの成績一覧
- 総合スコア（カウントアップ演出）
- ランク表示（S/A/B/Cが派手に表示）
- 「もう1回プレイ」ボタン
- SNSシェアボタン
